# API Design Standards

<!-- Bangumi-api-client API 设计规范，操作 src/api/ 目录时自动加载 -->

## Two-Layer Architecture

| Layer | Location | Role |
| ------- | ---------- | ------ |
| **Layer 1** (generated) | `src/generated/` | Auto-generated by `@hey-api/openapi-ts`. Contains raw SDK functions (`sdk.gen.ts`), all request/response types (`types.gen.ts`), and HTTP client infrastructure (`client/`, `core/`). |
| **Layer 2** (hand-written) | `src/api/` | High-level per-resource API classes. Each class wraps the generated SDK functions, providing a clean, resource-focused interface. |

### Golden Rule

NEVER edit `src/generated/` files manually. They are regenerated by `yarn generate` and any manual changes will be lost.

When the official Bangumi OpenAPI YAML is updated, run `yarn generate` to refresh Layer 1.

## API Class Pattern

Every resource module follows this exact pattern:

```ts
import type { ClientResult } from '../client.js';
import type { Client } from '../generated/client/index.js';
import type { Subject } from '../generated/types.gen.js';

export class SubjectAPI {
  public debug: boolean;

  constructor(
    private readonly client: Client,
    debug = false,
  ) {
    this.debug = debug;
  }

  /**
   * 根据条目 ID 获取条目详情。
   *
   * `GET /v0/subjects/{subject_id}`
   *
   * @param subjectId - 条目 ID（正整数）
   * @returns `data` — 完整 `Subject` 对象；不存在时返回 HTTP 404
   */
  async getSubjectById(subjectId: number): Promise<ClientResult<Subject>> {
    const result = await this.client.get<Subject>({
      url: '/v0/subjects/{subject_id}',
      path: { subject_id: subjectId },
    });
    if (this.debug) {
      // eslint-disable-next-line no-console
      console.log('[SubjectAPI.getSubjectById]', JSON.stringify(result.data, null, 2));
    }
    return result as unknown as ClientResult<Subject>;
  }
}
```

### Required Elements

- Constructor takes `client: Client` (required) and `debug = false` (optional)
- `public debug: boolean` property for per-instance debug logging
- Every method returns `Promise<ClientResult<T>>`
- Debug logging guarded by `if (this.debug)`
- `// eslint-disable-next-line no-console` on debug `console.log` calls
- Return cast via `result as unknown as ClientResult<T>` (necessary due to generated client types)

## JSDoc Convention

Every public method MUST have JSDoc with:

1. Summary line describing the endpoint
2. HTTP method and route in a separate paragraph (surrounded by backticks)
3. `@param` for every parameter with type hints
4. `@returns` describing the shape and edge cases (404, empty arrays, etc.)

```ts
/**
 * 全文搜索条目。
 *
 * `POST /v0/search/subjects`
 *
 * @param options         - 搜索参数
 * @param options.keyword - 搜索关键词（必填）
 * @param options.sort    - 排序方式：`match`=相关度 | `heat`=热度 | `rank`=排名 | `score`=评分
 * @param options.limit   - 每页条数（默认 25，最大 25）
 * @param options.offset  - 分页偏移（默认 0）
 * @returns `data.total` — 符合条件的总数；`data.data` — 当页条目列表
 */
```

### JSDoc Rules

- Docs in Chinese (project convention)
- Nested params use dot notation: `@param options.sort`
- Enums in `@param` use backtick-wrapped values: `` `match` | `heat` ``
- Always document HTTP status edge cases in `@returns`

## `ClientResult<T>` Pattern

All API methods return `Promise<ClientResult<T>>`:

```ts
interface ClientResult<T> {
  data: T | undefined;
  error: unknown;
  response: Response;
  request: Request;
}
```

### Consumption Pattern

```ts
const result = await client.subjects.getSubjectById(374791);
if (result.error !== null && result.error !== undefined) {
  // handle error
  return;
}
// result.data is typed as Subject
console.log(result.data.name);
```

### Design Rationale

- Never throw from API methods — errors are returned, not thrown
- `error` is `unknown` — consumer must type-narrow it
- `response` and `request` are always available for diagnostics

## Type Organization

### Option Interfaces

For methods with 3+ parameters, define an options interface:

```ts
export interface SearchSubjectsOptions {
  keyword: string;
  sort?: 'match' | 'heat' | 'rank' | 'score';
  filter?: SearchSubjectsFilter;
  limit?: number;
  offset?: number;
}
```

### Result Interfaces

For methods that return a shaped response (pagination wrappers, etc.):

```ts
export interface SearchSubjectsResult {
  total: number;
  limit: number;
  offset: number;
  data: Subject[];
}
```

### Re-export Rules

- Type-only re-exports MUST use `export type` in `src/index.ts`
- `src/generated/types.gen.ts` types are re-exported via `export type * from`
- `src/generated/sdk.gen.ts` functions are re-exported via `export * from`

```ts
// src/index.ts
export { SubjectAPI } from './api/01-subjects.js';
export type {
  SearchSubjectsOptions,
  SearchSubjectsResult,
} from './api/01-subjects.js';
export type * from './generated/types.gen.js';
export * from './generated/sdk.gen.js';
```

## Module Organization

### File Naming

Files in `src/api/` use zero-padded kebab-case for ordering:

```text
src/api/01-subjects.ts      # SubjectAPI — 8 endpoints
src/api/02-episodes.ts      # EpisodeAPI — 2 endpoints
src/api/03-characters.ts    # CharacterAPI — 7 endpoints
src/api/04-persons.ts       # PersonAPI — 7 endpoints
src/api/05-users.ts         # UserAPI — 3 endpoints
src/api/06-collections.ts   # CollectionAPI — 12 endpoints
src/api/07-revisions.ts     # RevisionAPI — 8 endpoints
src/api/08-indices.ts       # IndexAPI — 9 endpoints
```

### File Size

- Max 500 lines per file
- If a module exceeds 500 lines, split into sub-modules (e.g., `01-subjects-calendar.ts`, `01-subjects-search.ts`)

## Client Factory

`src/client.ts` exports `createBangumiClient()`:

```ts
const client = createBangumiClient({ token: 'your-access-token' });
```

### Options

| Option | Type | Default | Description |
| -------- | ------ | --------- | ------------- |
| `token` | `string \| undefined` | `undefined` | Bangumi Access Token for authenticated requests |
| `baseUrl` | `string \| undefined` | `'https://api.bgm.tv'` | API base URL |
| `userAgent` | `string \| undefined` | `'Bangumi-api-client/{VERSION} (...)'` | User-Agent header value |

### Auth Injection

When `token` is provided and non-empty, the factory adds `Authorization: Bearer {token}` to all requests automatically.

## Error Handling

### Never Throw

API methods never throw. All errors (network, HTTP 4xx/5xx, parse errors) are captured in `result.error`.

### Debug Logging

Each API class has a `debug` flag. When `true`, `console.log` outputs the full response data. Debug logs are disabled by default.

```ts
const api = client.subjects;
api.debug = true; // enable debug logging at runtime
```

## Integration Testing

Each API module has a corresponding integration test:

```text
tests/integration/01-subjects.test.ts
tests/integration/02-episodes.test.ts
...
```

Tests require network access to `api.bgm.tv`.
