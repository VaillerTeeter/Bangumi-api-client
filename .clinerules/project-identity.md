## Project Name & Purpose

**Bangumi-api-client** is a TypeScript client library that wraps all 56 endpoints of the [Bangumi API v0](https://github.com/bangumi/api), providing type-safe, high-level calling interfaces across 8 resource modules: Subject, Episode, Character, Person, User, Collection, Revision, and Index.

## Tech Stack

### Core

- **TypeScript 5.x** — ESM-first (`"type": "module"`), ES2022 target, `NodeNext` module resolution
- **@hey-api/openapi-ts** — code generation from official Bangumi OpenAPI YAML spec (dev dependency)
- **@hey-api/client-fetch** — generated HTTP client runtime (runtime dependency)
- **Vitest** — test runner with full ESM compatibility

## Architecture

### Two-Layer Architecture

| Layer | Location | Description |
| ------- | ---------- | ------------- |
| **Layer 1** (generated) | `src/generated/` | Auto-generated by `@hey-api/openapi-ts` from official OpenAPI YAML. Contains raw SDK functions (`sdk.gen.ts`), all request/response types (`types.gen.ts`), and HTTP client infrastructure (`client/`, `core/`). NEVER edit manually. |
| **Layer 2** (hand-written) | `src/api/` | High-level per-resource API classes (e.g., `SubjectAPI`, `EpisodeAPI`). Each class wraps the generated SDK functions, providing a clean, resource-focused interface with `ClientResult<T>` return types. |

### Client Factory

`src/client.ts` exports `createBangumiClient()` — a factory function that initializes the generated client with optional `accessToken` for authentication (`Authorization: Bearer`) and user-specified `User-Agent`.

### Public API

`src/index.ts` re-exports all 8 API classes, the client factory, and key types. Consumers import from the package root.

## Release Workflow

1. Update `version` in `package.json`
2. Commit: `git add package.json && git commit -m "chore: bump version to <version>"`
3. Tag: `git tag v<version>`
4. Push: `git push origin HEAD && git push origin v<version>`
5. CI (`release.yml`) auto-publishes to npm on `v*` tag push
